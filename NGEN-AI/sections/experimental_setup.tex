\section{Experimental Setup}
\label{sec:setup}

\subsection{Models}

We benchmark seven generation models spanning different sizes, architectures, and deployment configurations. The selection covers dense transformers, mixture-of-experts architectures, and both local and cloud-hosted deployments, allowing us to assess how model scale and serving infrastructure interact with RAG pipeline quality.

\begin{itemize}
  \item \textbf{Qwen3-32B} --- A 32B dense transformer, served locally on an RTX 5090 with 4-bit AWQ quantization via Ollama.
  \item \textbf{Qwen3.5-397B-A17B} --- A MoE model with 397B total and 17B active parameters~\cite{qwen2025qwen3}. Served via vLLM on an H100 cluster.
  \item \textbf{GPT-OSS-20B} --- A 20B open-source MoE model (5B active)~\cite{gptoss2024}, served locally via Ollama on an NVIDIA DGX Spark.
  \item \textbf{Qwen3.5-think} --- A reasoning variant of Qwen3.5 with chain-of-thought traces, served via vLLM on an H100 cluster.
  \item \textbf{GPT-4o-mini} --- OpenAI's compact frontier model via API, included as an upper-bound reference.
  \item \textbf{DeepSeek-V4-Pro} --- DeepSeek's latest flagship model, served via API with similarity retrieval only.
  \item \textbf{Qwen3.6-27B} --- A 27B model with NVFP4 quantization, served locally on an RTX 5090 via vLLM.
\end{itemize}

The critic LLM for RAGAS evaluation is Qwen3.5-397B-A17B in non-thinking mode, running on the same H100 cluster. Using a separate, capable critic model avoids self-evaluation bias and ensures consistent quality judgments across all tested generation models.

\subsection{Embedding and Dataset}

All configurations use \texttt{nomic-embed-text:latest}~\cite{nomic2024nomic} as the embedding model, served via Ollama. This model achieves competitive results on the MTEB benchmark~\cite{muennighoff2022mteb} with a dimensionality of 768, making it well-suited for semantic similarity tasks in the retrieval stage.

We use the Stanford Question Answering Dataset (SQuAD)~\cite{rajpurkar2016squad} in its default configuration. Each sample contains a question, a ground-truth answer, and a context paragraph drawn from Wikipedia articles. We sample 100 questions per configuration. The shared corpus of context paragraphs is loaded once and reused across all configurations, ensuring that differences in retrieval quality stem solely from the chunking and embedding parameters rather than from dataset variation.

\subsection{Prompt Templates}

Two prompt templates are evaluated, representing opposite ends of the instruction spectrum. The \textbf{Concise} template provides a minimal instruction that asks the model to answer the question based on the provided context and keep the response brief and factual. The \textbf{Detailed} template instructs the model to provide a thorough answer with supporting evidence from the context, include relevant details, and maintain strict factual accuracy. This contrast allows us to measure how instruction specificity affects faithfulness and answer completeness across models of different sizes.

\subsection{Evaluation Protocol}

RAGAS metrics (faithfulness, context recall, semantic similarity) are computed using Qwen3.5-397B-A17B as the critic model with a maximum of 4096 critic tokens per evaluation call. IR metrics (Hit@k, nDCG@k, Recall@k for $k \in \{1, 3, 5\}$) are computed against gold document labels provided by the SQuAD dataset. NLG metrics use RoBERTa-large for BERTScore~\cite{zhang2020bertscore} and standard implementations for METEOR~\cite{banerjee2005meteor}, ROUGE-L~\cite{lin2004rouge}, and BLEU~\cite{papineni2002bleu}. All metrics are reported as means over the 100-question sample, with per-sample standard deviations available in the supplementary results.
