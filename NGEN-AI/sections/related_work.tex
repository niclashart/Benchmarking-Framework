\section{Related Work}
\label{sec:related}

RAG was introduced by Lewis et al.~\cite{lewis2020retrieval}, combining dense passage retrieval~\cite{karpukhin2020dense} with sequence-to-sequence generation. Subsequent systems extend retrieval into pre-training, self-reflection, dynamic retrieval, graph-based reasoning, and evidence checking~\cite{guu2020realm,asai2024selfrag,jiang2024active,edge2024local,li2024chain}. Surveys~\cite{gao2024retrieval,zhao2024retrieval} show that chunking, retrieval, and prompt design affect output quality, but offer no tooling for systematic configuration comparison.

Several frameworks evaluate a \emph{single} pipeline configuration. RAGAS~\cite{es2023ragas} introduced LLM-based metrics (faithfulness, answer relevancy, context precision/recall). ARES~\cite{saadfalcon2024ares} uses prediction-powered inference without gold references. RAGChecker~\cite{ragchecker2024} decomposes performance into retrieval/generation diagnostics. Chen et al.~\cite{chen2024benchmarking} benchmark LLMs on RAG but evaluate only model-level differences. Beyond metric suites, observability platforms (LangSmith~\cite{langsmith2024}, Langfuse~\cite{langfuse2024}) provide tracing and cost telemetry without cartesian-product sweeps or IR/NLG metrics; TruLens~\cite{trulens2023} offers the RAG Triad for single-config evaluation; DeepEval~\cite{deepeval2024} adapts pytest-style workflows for LLM regression tests. None systematically sweep the multi-component configuration space.

Individual components have been studied in isolation: lost-in-the-middle effects~\cite{liu2024lost} motivate chunking/retrieval strategies; MMR~\cite{carbonell1998mmr} balances relevance and diversity; MTEB~\cite{muennighoff2022mteb} benchmarks embedding models; Sentence-BERT~\cite{reimers2019sentence} enables efficient semantic similarity. These studies examine one component at a time, leaving interaction effects across the full pipeline unexplored.

In the broader NLP ecosystem, benchmarks like GLUE, SuperGLUE, and MTEB~\cite{muennighoff2022mteb} standardize task-specific evaluation. RAG benchmarks are typically dataset-centric (SQuAD~\cite{rajpurkar2016squad}, MS MARCO~\cite{wu2024msmarco}, medical RAG~\cite{song2024bench}) rather than configuration-centric. \toolname{} fills this gap by treating configuration exploration as a first-class concern: given parameter ranges for all pipeline components, it generates, executes, evaluates, and tracks every combination.
